UX Design

Concerning UX and my approach

Being a full-stack designer means that whenever possible, I like to design complete experiences. Experiences that can take users seamlessly from one device to another. I like to start by talking about how to make important, upfront planning decisions. Then use context as a guide to create a flexible content strategy with particular emphases on design for touch screens; one of the most important interfaces in today's world.

A quick run-through of my preferred workflow goes something like this:

  1. Get a fix on the primary and secondary user groups Iâ€™m designing for, and create at least 2 or 3 user personas.
  2. Hash out a content survey document that classifies and prioritizes the website or app content (both visual a written). This helps me and other team members get a handle on how content flow should change across multiple devices.
  3. Create wireframes to define the overall content flow and user routs as they would likely navigate through the site or app.
  4. Create the mood boards (also known as design style guides).
  5. Create the site/app/system layout and user interface (UI).
  6. Once finalized, I start writing the front-end code.
